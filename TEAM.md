## Team Communicatie

### Standaard Werkwijze
- **Daily Standups**: Dagelijks, 15 minuten
- **Sprint Planning**: Elke 2 weken
- **Sprint Review**: Eind van elke sprint
- **Retrospectives**: Na elke sprint
- **Code Reviews**: Verplicht voor alle PRs

### Tools
- **Project Management**: GitHub Projects
- **Code Repository**: GitHub
- **Communication**: Slack/Discord
- **Design**: Figma
- **Documentation**: Markdown in repo

### Besluitvorming
- **Architecture Decisions**: Lead Frontend + Product Owner
- **Design Decisions**: UX Designer + Product Owner
- **Technical Decisions**: Engineering team consensus
- **Priority Decisions**: Product Owner (final say)

## Verantwoordelijkheid Matrix (RACI)

| Activiteit | Product | Engineering | Design | QA | DevOps |
|-----------|---------|-------------|--------|----|---------|
| Feature Definition | A/R | C | C | I | I |
| UI/UX Design | A | C | R | C | I |
| Code Development | A | R | C | I | I |
| Testing | A | C | I | R | I |
| Deployment | A | C | I | C | R |
| Bug Fixes | C | R | I | A | C |

**R** = Responsible, **A** = Accountable, **C** = Consulted, **I** = Informed
